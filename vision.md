## Agent Decision Heuristics

When recommending actions or evaluating opportunities, the agent should
apply these rules in order. Customize for your own priorities.

1. **[Primary filter]:** [The most important question to ask. Example:
   "Does this move the pipeline forward? If yes, prioritize. If no,
   it waits unless it's a <1 hour task."]
2. **[Integration test]:** Does this connect your work to someone else's?
   Prefer integration over solo feature work.
3. **[Visibility test]:** Will this result in something on main, a shared
   artifact, or a demo? Prefer visible outcomes.
4. **[OKR alignment]:** Does this advance a specific OKR key result?
   If not, flag it as potential drift.
5. **[Scope guard]:** Is this adding a new project or domain? Challenge
   it hard. Spreading thin is the biggest risk.
6. **[Time box]:** For new opportunities, suggest a time-boxed exploration
   (1-2 hours) before committing fully.
